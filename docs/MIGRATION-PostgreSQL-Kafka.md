# CouponKill 技术栈迁移文档：MySQL → PostgreSQL，RocketMQ → Kafka

> 本文记录本次迁移的范围、逐层改动、运行方式、验证结果与遗留事项。
>
> **重要澄清**：需求描述为“RabbitMQ 换成 Kafka”，但代码库中**不存在 RabbitMQ**，实际业务消息中间件是 **RocketMQ**。因此本次迁移的真实目标是 **RocketMQ → Kafka**（已与需求方确认）。

---

## 1. 迁移目标与范围

| 维度 | 迁移前 | 迁移后 |
|------|--------|--------|
| 关系型数据库 | MySQL 8.0 | PostgreSQL 16 |
| JDBC 驱动（Java） | `mysql-connector-j` / `mysql-connector-java` | `org.postgresql:postgresql` |
| ORM / 分库分表 | MyBatis + ShardingSphere 5.5.2 | 不变（驱动/URL/方言改写，分片规则保留） |
| Go 数据库驱动 | `github.com/go-sql-driver/mysql` | `github.com/lib/pq` |
| 消息中间件 | RocketMQ 4.9.4（`rocketmq-spring-boot-starter`） | Apache Kafka（`spring-kafka`） |
| 消息序列化 | RocketMQ 默认 Jackson JSON | Kafka `JsonSerializer`/`JsonDeserializer` |
| Nacos 元数据库 | MySQL | 本地内置 Derby（生产 PG 需插件，见 §7） |

覆盖模块：`couponkill-common`、`couponkill-user-service`、`couponkill-order-service`、`couponkill-coupon-service`、`couponkill-gateway`、`couponkill-go-service`，以及 Nacos 配置、Helm Chart、docker-compose。

## 2. 数据库迁移（MySQL → PostgreSQL）

### 2.1 统一 Schema 真源
迁移前存在三套字段不一致的 DDL（`init.sql`、`db-init-job.yaml` 内嵌、Mapper 实际字段）。本次**以 common 模块 POJO + MyBatis Mapper 为唯一真源**重建 PG schema。

新增权威建库建表脚本：**`charts/couponkill/scripts/init-postgres.sql`**

库表布局与 ShardingSphere 分片规则严格对齐：

| 服务 | 数据库 | 表 |
|------|--------|-----|
| user | `user_db_0` / `user_db_1` | `"user"`、`user_coupon_count` |
| order | `order_db_0` / `order_db_1` | `order_0 .. order_15` |
| coupon | `coupon_db_0` / `coupon_db_1` | `coupon_0 .. coupon_15`、`stock_log_0 .. stock_log_15` |

要点：
- MySQL `AUTO_INCREMENT` → PG `GENERATED BY DEFAULT AS IDENTITY`；`ENGINE=InnoDB` / `utf8mb4` 移除。
- `DATETIME`/`TIMESTAMP` → `TIMESTAMP`；`DECIMAL` → `NUMERIC`。
- MySQL `ON UPDATE CURRENT_TIMESTAMP` 无对应；应用层已显式写 `update_time`，故删除。
- `user` / `order` 为 PG 保留字，统一改用双引号 `"user"` / `"order"`。
- 分片表用 `DO $$ ... FOR i IN 0..15 ... $$` 循环创建。

### 2.2 Maven 依赖
- 父 `pom.xml`：`<mysql.version>` → `<postgresql.version>42.7.4`；`mysql-connector-java` → `org.postgresql:postgresql`。
- `couponkill-common/pom.xml`：`mysql-connector-java` → `postgresql`；**新增 `com.google.guava:guava`**（`RedisCache` 的布隆过滤器此前经 RocketMQ 传递引入，去掉 RocketMQ 后需显式声明）。
- 三个服务 `pom.xml`：`com.mysql:mysql-connector-j` → `org.postgresql:postgresql`。

### 2.3 ShardingSphere 数据源（Nacos `nacos/shard/DEFAULT_GROUP/*.yaml`）
- `driverClassName: com.mysql.cj.jdbc.Driver` → `org.postgresql.Driver`
- `jdbc:mysql://mysql:3306/xxx?charset=utf8mb4&parseTime=...` → `jdbc:postgresql://postgres:5432/xxx?reWriteBatchedInserts=true&ApplicationName=...`
- 账号 `root/root` → `postgres/postgres`
- 说明：文件中被注释的集群块与 `enabled:false` 的 `sharding.cluster` 仍含 `:3306`，为**惰性历史配置**，未启用，未改动。

### 2.4 MyBatis Mapper 方言改写
| 文件 | 改动 |
|------|------|
| `UserMapper.xml` | 所有 `` `user` `` 与裸 `user` → `"user"`；`ON DUPLICATE KEY UPDATE` → `ON CONFLICT (user_id) DO UPDATE SET ... version = user_coupon_count.version + 1` |
| `OrderMapper.xml` | 13 处 `` `order` `` → `"order"`（`NOW()`/`LIMIT` PG 兼容，保留） |
| `CouponMapper.xml` | `DATE_SUB(NOW(), INTERVAL valid_days DAY)` → `NOW() - (valid_days * INTERVAL '1 day')`；`SELECT * ... GROUP BY id, shard_index` 依赖 PG 主键函数依赖，合法，保留 |
| `UserCouponCountMapper.java` | 去反引号；`ON DUPLICATE KEY UPDATE` → `ON CONFLICT (user_id) DO UPDATE SET ...`（用 `user_coupon_count.col + #{...}`） |

### 2.5 唯一约束异常识别（`OrderServiceImpl.isDuplicateEntryException`）
新增 PostgreSQL 识别：`duplicate key value violates unique constraint` / SQLState `23505`（保留历史 MySQL/SQLite 文案兼容）。

## 3. 消息中间件迁移（RocketMQ → Kafka）

### 3.1 依赖
父 POM 与各模块的 `rocketmq-client`/`rocketmq-remoting`/`rocketmq-spring-boot-starter`/`spring-cloud-starter-stream-rocketmq` → **`org.springframework.kafka:spring-kafka`**（版本由 `spring-boot-dependencies` BOM 管理）。

### 3.2 公共配置
- 删除 `couponkill-common/.../RocketMQBaseConfig.java`
- 新增 `couponkill-common/.../KafkaBaseConfig.java`：`ProducerFactory` + `KafkaTemplate<String,Object>`，key=String、value=`JsonSerializer`，`acks=all`、幂等开启、`ADD_TYPE_INFO_HEADERS=false`（消费端按目标类型反序列化，避免跨服务包名耦合）。

### 3.3 生产者
| 位置 | 改动 |
|------|------|
| `OrderServiceImpl` | `RocketMQTemplate` → `KafkaTemplate<String,Object>`；`convertAndSend` / `sendMessageInTransaction` → `kafkaTemplate.send(topic, key, msg)`；`@Value("${rocketmq.topic.*}")` → `${kafka.topic.*}` |
| `SeckillService`（coupon） | 同上；以 `reqId` 作为消息 key |

> 原 RocketMQ 事务消息的本地事务回查恒返回 `COMMIT`（无实际业务逻辑），故简化为普通发送。

### 3.4 消费者
- 新增 `couponkill-order-service/.../config/KafkaConsumerConfig.java`：面向 `OrderMessage` 的 `ConcurrentKafkaListenerContainerFactory`（`JsonDeserializer(OrderMessage)`，信任 `com.aliyun.seckill.common.pojo`）。
- `SeckillOrderResultListener`：`@RocketMQMessageListener` → `@KafkaListener`（topic/group 走占位符，`containerFactory=orderMessageKafkaListenerContainerFactory`）。
- 删除 `OrderTransactionListener.java`、`RocketMQTransactionListenerConfig.java`。

### 3.5 顺带修复
`CouponServiceImpl`：`javax.annotation.PostConstruct` → `jakarta.annotation.PostConstruct`（Spring Boot 3 下 `javax` 由 RocketMQ 传递引入，去掉后失效；`jakarta` 才被 Spring 6 处理）。

## 4. Go 服务迁移（couponkill-go-service）

方言级迁移（保留包名/类型名与既有分片逻辑）：
- `go.mod`：`go-sql-driver/mysql` → `github.com/lib/pq v1.10.9`（`go mod tidy` 已更新 `go.sum`）。
- `sql.Open("mysql", ...)` → `sql.Open("postgres", ...)`（`mysql_client.go` ×2、`sharding_datasource.go` ×1）。
- `mysql_repo.go`：占位符 `?` → `$1..$n`（7 条 SQL）；去反引号表名；`ON DUPLICATE KEY UPDATE ... VALUES(col)` → `ON CONFLICT (user_id) DO UPDATE SET ... EXCLUDED.col`；`isDuplicateEntryError` 改为 `pq.Error.Code == "23505"`。
- `db_config.go`（死代码）：`@tcp(...)` DSN → PG `host=... port=... user=... password=... dbname=... sslmode=disable`。
- Nacos `go-service-dev.yaml`：`mysql.dsn`（key 保留为历史名）值改为 PG DSN。

## 6. 部署与本地联调

### 6.1 新增：`docker-compose.migration.yml`（已验证可运行）
一键起本地基础设施：
```bash
docker compose -f docker-compose.migration.yml up -d
# 停止并清理数据卷
docker compose -f docker-compose.migration.yml down -v
```
- **postgres:16**：自动执行 `init-postgres.sql` 建库建表（`5432`）
- **apache/kafka:3.8.0**：KRaft 单节点（`9092`），`kafka-init` 建 3 个 topic（16 分区）
- **redis:7**（`6379`）
- **nacos/nacos-server:v3.1.1**：standalone（内置 Derby；与 nacos-client 3.1.1 对齐）

### 6.2 Helm Chart（`charts/couponkill`）主链路
- `dependencies.yaml`：MySQL Service（3306）→ PostgreSQL Service（5432）
- `values.yaml`：`db` 块 → PG；新增 `kafka` 段；`rocketmq.enabled: false`；`mysql.enabled: false`（key 保留兼容）；`nacos.storage.type: embedded`
- `db-init-job.yaml`：重写为 `postgres` 镜像 + `psql`，通过 ConfigMap（`.Files.Get "scripts/init-postgres.sql"`）执行权威 DDL
- `deploy-user.yaml`：`SPRING_DATASOURCE_URL` → `jdbc:postgresql://postgres:5432/...`
- `deploy-go.yaml`：`MYSQL_DSN` → PG DSN
- `deploy-order.yaml`/`deploy-go.yaml`：已有 `{{ if .Values.rocketmq.enabled }} ROCKET_MQ {{ else }} KAFKA {{ end }}`，RocketMQ 关闭后自动注入 `KAFKA`
- `nacos-init-job.yaml`：驱动类 → `org.postgresql.Driver`；`common`/`middleware` 的 rocketmq 内嵌块 → kafka
- **删除** `rocketmq.yaml`、`rocketmq-init-job.yaml`（Kafka Service 与 `kafka-init-job.yaml` 已存在）
